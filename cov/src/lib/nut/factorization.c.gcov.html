     757             :  * Montgomery form is by^2=x^3+ax^2+x.  Aside from what form we use for the curve, there are many possible ways to store points on the curve.
     758             :  * The simplest is to simply store x and y and a boolean indicating if the point is actually identity or not.  However, this creates branching
     759             :  * and complexity due to handling points at infinity and finite points on the curve separately.  So instead we can use projective coordinates,
     760             :  * where x and y are expressed as ratios x=X/Z and y=Y/Z.  If Z is zero, we have a point at infinity.  Variations of this exist where X and Y are
     761             :  * divided by different powers of Z.
     762             :  *

     764             :  * For any y, there are at most 3 solutions x putting it on the curve.  So it's pretty concievable that X and Z might be sufficient to compute
     765             :  * the x coordinate of any scalar multiple of a point on the curve.  In fact, it turns out this is true.  Using projective coordinates also lets
     766             :  * us do fewer gcd computations, hence the motivation to have a Z coordinate, yet we do not really need the y coordinate per se, so it is great
